Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The ul element is an unordered list (bullet points)
  • The "title" within head is displayed in the browser
  • "alt" is a text description of the img element it is attached too
  • "meta" within the head section indicates characterastics of the webpage
  • class and ID can be used with css to avoid repeat inline code

CSS

  • Cascading style sheets notes
  • define style characterastics such as font type, size, color or even margins
  • border-radius can defuse the boxy nature that elements are contained in
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • a class selector can be defined and then attributed to all elements that are linked to it
  • can be inline, within the head or linked via a css file
  • box shadow elements, first px value is horizontal, 2nd is vertical and 3rd is blur (this can also be inset)

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • providing the branch has not been deleted in github prompt, this can be used repeatedly for each edit
  • git has a lot of commands i need to put into practice before it sinks in
